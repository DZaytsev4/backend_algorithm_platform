# Класс UserViewsTests
1) **test_register_user_success**
Что проверяет: Успешную регистрацию нового пользователя в системе
Как делает: Отправляет POST-запрос на endpoint регистрации с валидными данными (username, email, пароли), затем проверяет что:

Возвращается статус 201 Created

В ответе присутствуют корректные username и email

Пользователь действительно создан в базе данных

2) **test_register_user_password_mismatch**
Что проверяет: Валидацию несовпадающих паролей при регистрации
Как делает: Отправляет POST-запрос с разными значениями в полях password и password2, затем проверяет что:

Возвращается статус 400 Bad Request

В ответе присутствует сообщение об ошибке для поля password

3) **test_register_user_duplicate_username**
Что проверяет: Защиту от регистрации с уже существующим username
Как делает: Пытается зарегистрировать пользователя с username, который уже используется, затем проверяет что:

Возвращается статус 400 Bad Request

В ответе присутствует сообщение об ошибке для поля username

4) **test_register_user_duplicate_email**
Что проверяет: Защиту от регистрации с уже существующим email

5) **test_get_current_user_authenticated**
Что проверяет: Доступ к данным текущего пользователя для аутентифицированных пользователей

Возвращается статус 200 OK

В ответе присутствуют корректные данные пользователя (username, email)

6) **test_get_current_user_unauthenticated**
